gerrit with git

  • 安装git工具
  • 配置git
  • 生成SSH Keys
  • 向Gerrit添加SSH keys
  • Git于Gerrit配合使用

###一、 安装git工具

1.Linux 
$ sudo yum install git 或者 $ sudo apt-get install git
2.Windows
Git官网下载.exe 文件安装即可

二、配置git

1.查看配置变量
$ git config -l
core.symlinks=false
core.autocrlf=true
core.fscache=true
color.diff=auto
color.status=auto
color.branch=auto
color.interactive=true
help.format=html
rebase.autosquash=true
http.sslcainfo=E:/Development tool/Git/git/Git/mingw64/ssl/certs/ca-bundle.crt
credential.helper=manager
filter.lfs.required=true
filter.lfs.clean=git-lfs clean -- %f
filter.lfs.smudge=git-lfs smudge -- %f
filter.lfs.process=git-lfs filter-process
user.name=daxiongit
user.email=daxiongit@qq.com
usr.email=daxiongit@qq.com
core.editor=emacs
core.autocrlf=false

2.配置用户名和邮箱
$ git config --global user.name "daxiongit"
$ git config --global user.email "daxiongit@qq.com"

三、生成 SSH keys

1.SSH keys
SSH 表示Secure Shell 或有时用于从远程计算机安全访问网络服务的Secure Socket Shell 协议，可以设置SSH密钥用来在计算机和Gerrit之间提供可靠的安全连接。
2.SSH 密钥文件位置
$ ls ~/.ssh/
id_rsa  id_rsa.pub
### id_rsa: 私有密钥
### id_rsa.pub: 公有密钥
3.生成新的SSH密钥
$ ssh-keygen -t rsa -C "daxiongit@qq.com"

四、向Gerrit 添加SSH密钥

1.复制id_rsa.pub 内容并复制下来

2.注册Gerrit 账户

[https://wikitech.wikimedia.org/wiki/Special:CreateAccount]

3.登录Gerrit

[http://gerrit.wikimedia.org/r/login/%23%2Fq%2Fstatus%3Aopen]

4.添加 SSH Public Keys

接着在右上角，点击您的用户名，然后选择设置(Settings)选项。在这里，我们创建了一个名为Daxiongit的帐户，以用来演示Gerrit的使用和演示。

###五、Gerrit配合Git使用

1.打开Git Bash获取 ssh-agent
$ eval 'ssh-agent'
2.将ssh密钥添加到ssh-agent
$ ssh-add ~/.ssh/id_rsa
！！！此时出现错误：
$ ssh-add ~/.ssh/id_rsa
Could not open a connection to your authentication agent.
！！！解决方案：
$ ssh-agent bash
$ ssh-add ~/.ssh/id_rsa

参考：【https://www.yiibai.com/gerrit/gerrit_overview.html】

Java高级开发（内存模型）

内存模型

内存空间(Runtime Data Area)按照是否线程共享分为两块：

线程共享

方法区（Method Area）和堆（Heap）

线程独享

Java虚拟机栈（Java Stack）、本地方法栈（Native Method Stack）、PC寄存器（Program Counter Register）

Java Stack

每个线程有一个私有栈，随着线程的创建而创建，栈中存放一个“栈桢”的东西，每个方法会创建一个“栈桢”，“栈桢”中存放了局部变量、操作数栈、方法出口等信息。

Native Method Stack

Native方法相关

Program Counter Register

PC寄存器（程序计数器），JVM支持多个线程同时运行，每个线程都有自己的程序计数器，

Heap

堆内存是JVM所有线程共享的部分，在虚拟机启动的时候就已经创建。所有的对象和数组都在堆上进行分配。这部分空间可通过 GC进行回收。

原子性

原子性指一个操作是不可中断的，即使是在多个线程并发执行的时候，一个操作一旦开始，就不会被其它线程干扰。

有序性

在并发时，程序的执行可能会出现乱序，指令会出现重排。

可见性

可见性是当一个线程修改了某一个共享变量的值时，其他线程是否立即直到这个修改。

Happen-Before规则

程序顺序原则：一个线程内保证语义的串行性

volatile规则：volatile变量的写发生在读之前，保证了volatile变量的可见性。

锁规则：解锁放生在加锁之后

传递性：A先于B，B先于C，A必然先于C

线程的start()方法先于它的每一个动作

线程的所有操作先于线程的终结(Thread.join())

线程的中断(interrupt()只是修改中断标识) 先于被中断线程的代码

对象的构造函数执行结束先于finalize()方法

Java高级开发（并发基础）

线程基础操作

new start runnable terminated blocked waiting time_waiting

1.终止线程（废除）

Thread.stop() // 不推荐使用，线程不安全

2.中断线程

(1)void Thread.interrupt() // 中断线程
(2)boolean Thread.isInterrupted() // 判断是否中断线程，不会清除当前中断状态
(3)boolean Thread.interrupted() // 判断线程是否被中断，并清除当前中断状态
(4)一个线程可以调用另一个线程的interrupt()方法来试图中断另一个线程的运行，也可以中断自己，如果线程没有运行，则interrupt()不会有任何影响，interrupt()只是做一个标记，需要线程自己来处理是否停止。
(5)如果被中断的线程处于wait(),join(),sleep()状态，那么该线程会立即接受一个InterruptedException异常，该异常会自动清除线程的中断状态。

3.挂起(suspend)和继续执行(resume)线程(废除)

线程suspend()之后才能执行resume(),否则会导致死锁

4.yield() 谦让

使线程从运行状态转化为就绪状态，也就是将资源释放出来，但是释放完后我也会和其它线程公平竞争释放出来的资源，这时有可能别的线程抢到了资源，也有可能自己抢到了资源。

5.join()

将一个线程加入到其他线程一起执行。也就是如果一个线程调用 join() 方法，那么其他线程会等待该线程执行完然后在一起执行后面的操作。

6.守护线程(Daemon)

在后台默默地完成一些系统性的服务，比如垃圾回收线程，JIT线程就可以理解为守护线程。在一个Java应用内，只有守护线程时，Java虚拟机就会自然退出。

7.线程优先级

8.wait() / notify()

wait() 等待直到被通知。
wait() 和 notify() 是java.lang.Object 上的方法，并不直接来源于Thread.
这两个方法都必须在该线程获取锁的情况下才能使用，也就是必须从同步环境内调用wait(),notify(),notifyAll()方法。
wait之后，该线程会立即释放同步Object的锁，并转入阻塞状态。
一个notify只能随机唤醒一个wait的线程，而notifyAll可以唤醒所有。
