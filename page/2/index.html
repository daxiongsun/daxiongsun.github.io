虚拟机模型

  1. 类加载子系统
    负责从文件系统或者网络中加载Class 信息(.class文件)，加载的信息存放到方法区中。
  2. 方法区
    方法区用于存放类信息、常量信息、常量池信息、字符串字面量和数字常量等。
  3. java堆
    java虚拟机在启动的时候会建立java堆，这个java堆就是程序主要的内存工作区域，用于存放对象实例，堆空间是所有线程共享的区域。
  4. 直接内存
    用于提高性能的区域，直接内存的速度优于java堆，适合读写频繁的场合，e.g. Java的NIO库允许java程序使用直接内存。
  5. java栈
    每个虚拟机线程都有一个私有的栈，在创建线程的时候，会创建一个私有栈，这个java私有栈中存放局部变量、方法参数、方法调用和方法返回值等。
  6. 本地方法栈
    本地方法栈用于本地方法调用，java虚拟机允许java直接调用本地方法(c,c++)
  7. 垃圾收集系统
    java 有一套完善的垃圾收集机制，用于回收无用对象、变量等
  8. PC寄存器
    PC寄存器是每个线程私有的空间，用于记录当前执行环境指针、程序计数器、操作栈指针、变量指针等。
  9. 执行引擎
    执行引擎负责虚拟机的字节码编译成机器码并执行。
